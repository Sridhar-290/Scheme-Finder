# AI Public Benefits Navigator

A full-stack web application that helps Indian citizens discover and apply for government schemes they're eligible for. Uses AI-powered chat and a multi-step eligibility checker to surface personalized benefits.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 5000)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string
- Optional env: `OPENAI_API_KEY` — enables real AI responses in chat (falls back to smart mock when absent)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite, Tailwind CSS, shadcn/ui, wouter routing, TanStack Query
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec → React Query hooks + Zod schemas)
- Build: esbuild (CJS bundle)
- AI: OpenAI `gpt-4o-mini` (when key set), smart mock fallback

## Where things live

- `lib/api-spec/openapi.yaml` — source of truth for all API contracts
- `lib/db/src/schema/` — Drizzle DB schema (users, schemes, savedSchemes, conversations, messages)
- `artifacts/api-server/src/` — Express routes, services
- `artifacts/api-server/src/services/ai.ts` — AI chat + eligibility scoring service
- `artifacts/navigator/src/pages/` — All frontend pages
- `artifacts/navigator/src/index.css` — Design tokens (navy primary, saffron secondary, green accent)

## Architecture decisions

- **Contract-first API**: OpenAPI spec → Orval codegen → typed React Query hooks + Zod validators. Never write API client code by hand.
- **Mock-first AI**: `ai.ts` detects `OPENAI_API_KEY` at startup; when absent it serves curated mock responses so the app works without any AI cost.
- **Route ordering**: `/api/v1/schemes/stats` and `/api/v1/schemes/featured` must stay BEFORE `/:id` in the schemes router.
- **Default user ID = 1**: No auth yet — all data scoped to a seeded user record.
- **Hooks from barrel only**: Always import from `@workspace/api-client-react`, never from deep generated paths.

## Product

- **Homepage**: Hero search, stats bar, category cards, featured schemes, how-it-works, testimonials, FAQ, footer
- **Browse Schemes** (`/schemes`): Searchable, filterable directory with category sidebar and save/unsave
- **Find Benefits** (`/find`): 5-step eligibility form → personalized results with eligible/partially-eligible schemes, document checklist, estimated benefits value
- **Ask AI** (`/ai`): ChatGPT-style interface with conversation sidebar, prompt chips, markdown rendering, typing indicator
- **Profile** (`/profile`): User details form — name, age, gender, state, income, occupation, social category
- **Saved Schemes** (`/saved`): Bookmarked schemes with sort and remove
- **Scheme Detail** (`/schemes/:id`): Full scheme info with save, share, and apply button

## Gotchas

- Always run `pnpm --filter @workspace/api-spec run codegen` after any change to `openapi.yaml` — it rebuilds both the React Query hooks AND the Zod validators simultaneously.
- `date-fns` is already in the navigator's dependencies — use it for date formatting.
- `openai` npm package is installed in `api-server` — the client is created conditionally on `OPENAI_API_KEY`.
